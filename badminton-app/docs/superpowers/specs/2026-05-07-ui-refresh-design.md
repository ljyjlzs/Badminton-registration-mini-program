# 羽毛球小程序 UI 统一优化设计

日期：2026-05-07
主题：全局界面优化，采用“清爽运动工具风”

## 1. 背景与目标

当前小程序已经具备较完整的业务能力，首页、活动列表、活动详情、创建活动、排名等页面也各自有一定样式基础。但从整体观感看，主要问题不是“没有设计”，而是“局部各自优化过，但整套视觉语言没有统一”。

当前问题集中在以下几点：

- 页面之间的卡片、按钮、标签、渐变使用规则不一致，整体观感偏散。
- 一些页面强调色过多，重要信息和普通信息没有稳定层级。
- 列表页、详情页、表单页的信息组织方式不同，用户需要重新适应每一页的视觉逻辑。
- 排名页、比赛页有竞技氛围，但在全局体系里偏跳脱，缺少与其他页面的统一关系。

本次优化目标不是重做业务流程，也不是追求炫技式改版，而是在保留现有产品结构和交互逻辑的前提下，建立一套统一、可持续扩展的视觉体系，让产品更像一个成熟的赛事管理工具。

本次优化的目标定义为：

- 保留运动氛围，但优先强化工具属性与信息可读性。
- 通过统一组件规则提升整体质感，而不是依赖单页堆砌装饰。
- 允许中等强度改版，包含部分首页区块层级和版式调整。
- 尽量不影响现有业务流程、页面跳转与核心交互。

## 2. 设计原则

本次视觉优化遵循以下原则：

### 2.1 真实问题优先

本次优化聚焦于已有页面的视觉统一、层级清晰和观感提升，不引入新的业务功能，不添加与当前场景无关的复杂装饰，不做与业务无关的大规模重构。

### 2.2 更简单的做法优先

优先通过统一变量、统一卡片体系、统一按钮规则、统一标签语义来解决“散”和“乱”的问题，而不是为每个页面单独定制一套风格。

### 2.3 向后兼容

本次改动只影响前端视觉层与必要的结构层级，不改动业务数据结构、云函数接口、页面跳转关系和核心状态机。

### 2.4 改一个地方时考虑全局

任何颜色、卡片、按钮、状态标签的调整都必须同步考虑首页、列表页、详情页、表单页、排名页和比赛页，避免出现“某一页更好看了，但整体更不统一”的问题。

## 3. 视觉方向

本次采用的视觉方向为“清爽运动工具风”。

这个方向的核心不是做成海报，也不是做成纯后台，而是让页面看起来像一个专业、轻快、适合频繁使用的赛事管理工具。

方向定义如下：

- 保留绿色作为品牌主色与操作主色。
- 背景和大多数容器采用更克制、更干净的中性色。
- 比赛、挑战、决赛、排名等高情绪场景保留有限度的竞技氛围。
- 普通页面强调清晰、轻盈、耐看，关键场景再释放情绪张力。

## 4. 全局视觉系统

### 4.1 配色策略

全局配色策略如下：

- 主色：沿用当前绿色体系，作为品牌主色、主按钮、选中态、关键数字、进度强调色。
- 中性色：引入更稳定的暖白、浅灰、浅青灰、深灰文本体系，作为背景、分割、容器与说明信息的主基础色。
- 功能色：保留成功、警示、危险、信息等状态色，但收敛使用范围。
- 赛事强调色：挑战赛、决赛、排名前三等场景允许使用粉色、橙色、金色等强调色，但必须局部使用，不能蔓延为全局主题。

约束规则如下：

- 首页、列表页、表单页以绿色 + 中性色为主，不大面积混用多种渐变。
- 紫色、粉色、橙色等高饱和渐变主要保留给赛事节点和排名展示。
- 同一种业务语义在所有页面使用同一颜色逻辑，例如“主要操作”始终是绿色主按钮，“危险操作”始终是红色轻强调按钮。

### 4.2 背景层级

背景层级统一为三层：

- 页面底层背景：极浅暖灰或浅青灰，整体干净、不抢内容。
- 普通内容容器：白色卡片承载。
- 强调内容容器：带轻微渐变或更明显阴影的重点卡片，仅用于首页欢迎区、比赛卡片、排名前三等场景。

目标是让背景退后，让内容与操作成为页面中心。

### 4.3 字体与信息层级

字体层级统一为：

- 页面主标题：突出但不过度夸张，用于首页欢迎区、核心模块标题。
- 模块标题：用于区块头部、列表分组、详情模块。
- 卡片主信息：活动名称、用户名称、关键比分。
- 次信息：时间、地点、人数、补充描述。
- 弱信息：说明文案、提示文案、空状态辅助文字。

约束如下：

- 同一层级的字号、字重、颜色应尽量统一。
- 活动名称、比赛名称等核心信息必须明显强于时间地点等辅助信息。
- 提示文案不得与主信息竞争注意力。

### 4.4 圆角、阴影与边框

统一采用圆润但不软塌的卡片语言：

- 普通卡片采用中到大圆角。
- 强调卡片采用更大圆角与更明显阴影。
- 边框保持极浅，仅用于分隔和提升边界清晰度。
- 阴影不追求厚重悬浮感，而是追求轻、透、干净。

阴影使用规则统一后，不同页面的卡片才会看起来属于同一产品。

## 5. 组件体系

### 5.1 卡片体系

统一定义三类卡片：

#### 基础卡片

用于：

- 活动列表卡片
- 普通信息区块
- 表单分组区块
- 详情页模块

特征：

- 白底
- 轻阴影
- 浅边框
- 中大圆角
- 稳定的内边距

#### 重点卡片

用于：

- 首页欢迎区
- 关键快捷入口
- 关键提示区

特征：

- 比基础卡片更强的层级
- 可带轻微渐变或轻高光
- 可加入少量运动感装饰
- 不承载复杂信息堆叠

#### 赛事卡片

用于：

- 比赛对阵卡片
- 挑战赛与决赛卡片
- 排名前三样式

特征：

- 允许使用更强的色彩和氛围
- 信息布局强调比分、对阵和状态
- 在视觉上明显区别于普通管理信息

### 5.2 按钮体系

按钮收敛为四类：

#### 主按钮

用于主要提交、确认、进入关键流程。

样式原则：

- 绿色实底或绿色渐变
- 白色文字
- 明确的按压反馈

#### 次按钮

用于次级操作。

样式原则：

- 白底或浅灰底
- 绿色描边或浅色描边
- 文本清晰但不抢主按钮注意力

#### 危险按钮

用于删除、取消报名等风险操作。

样式原则：

- 红色文字或浅红背景
- 明显区别于主按钮
- 不使用与主操作相同的视觉重量

#### 赛事按钮

用于开始挑战赛、开始决赛等少数特殊场景。

样式原则：

- 允许使用局部橙色、粉色等强调色
- 必须仅在赛事节点使用
- 不能替代全局主按钮体系

### 5.3 状态标签体系

状态标签统一规则：

- 统一圆角、字号、字重、内边距。
- 仅通过颜色区分状态，不改变整体结构。
- 在首页、活动列表、详情页中表达一致。

状态映射保持稳定，例如：

- 报名中：绿色系
- 分组中：橙黄色系
- 比赛中：蓝色系
- 挑战赛：粉色系
- 决赛：金橙色系
- 已结束：灰色系

## 6. 页面级设计

### 6.1 登录页

登录页从“单一登录按钮页面”提升为“轻品牌欢迎页”。

结构调整如下：

- 上半部分为欢迎视觉区，包含品牌标题、核心文案、轻量运动装饰。
- 中间突出产品价值，例如报名、分组、比赛、排名的一体化能力。
- 下半部分为授权登录按钮与辅助说明。

设计目标：

- 让用户首次打开时感知到这是一个完整产品，而不是默认模板页。
- 保留运动氛围，但避免花哨背景影响按钮识别。

### 6.2 首页

首页调整为三段式结构：

- 欢迎卡区域
- 快捷入口区域
- 活动分区区域

具体设计如下：

- 顶部欢迎卡承载头像、昵称、简短欢迎语和少量产品氛围。
- 快捷入口统一成一组风格一致的操作卡，不再只是两个孤立功能块。
- “我创建的活动”“可参加的活动”“不可参加的比赛”三组区块统一标题样式、间距、空状态和卡片规则。

首页的主要目标是：

- 第一眼建立品牌和专业感。
- 第二眼帮助用户快速进入高频任务。
- 第三眼清楚看到当前活动分布。

### 6.3 活动列表页

活动列表页的重点是“扫一眼能判断是否值得点进去”。

设计调整如下：

- 搜索栏更精致，边界更清楚，与背景层级分离。
- Tab 区域保持简洁稳重，选中态更清晰。
- 列表卡片统一信息顺序：活动名称、状态、地点、时间、类型、人数进度。
- 进度展示和状态标签保持统一语义，不再出现页面内多种表达方式。

列表卡片目标：

- 核心信息 1 秒可读。
- 同类信息固定位置，降低扫读成本。
- 点击态与可用性表达更明确。

### 6.4 活动详情页

活动详情页从“内容堆叠”调整为“信息分组”。

模块划分如下：

- 活动概览卡
- 报名信息区
- 当前用户状态与操作区
- 组织者操作区
- 比赛安排区
- 排名入口区

设计目标：

- 活动基础信息集中展示，减少分散扫描。
- 用户自己相关的操作和状态优先可见。
- 组织者操作有明确边界，不与普通用户信息混杂。
- 比赛安排保持赛事氛围，但不破坏整页可读性。

### 6.5 创建活动页

创建活动页定位为“引导式表单页”。

优化方向如下：

- 表单分组更清晰，标题与字段关系更明确。
- 输入框、类型切换、选择器、提示区块统一视觉规则。
- 说明文字更克制，避免和字段内容争抢注意力。
- 提交按钮区保持明确、稳定、易点。

目标是让用户快速完成配置，而不是在页面中反复辨认每个控件的含义。

### 6.6 排名页

排名页保留竞技感，但整体更克制、更耐看。

调整方向如下：

- 前三名保留奖牌感和区别度，但减少与全局风格脱节的跳跃感。
- 普通名次列表统一成更简洁的工具型样式。
- 分数、排名变化、队伍成员信息层级更明确。
- 双打队伍展示继续保留队伍结构，但提升整洁度。

排名页的设计目标是：

- 第一眼看清名次。
- 第二眼看清分数和人员。
- 保持一定荣誉感，但避免整页过于喧闹。

### 6.7 比赛卡片与赛事场景

比赛卡片、挑战赛、决赛相关区块是少数允许强化情绪的区域。

优化原则如下：

- 对阵关系、状态、比分是视觉主轴。
- 待开始、比赛中、待确认、已确认等状态在样式上必须容易区分。
- 动画只保留少量必要反馈，例如比赛中轻微脉冲、待确认轻微提醒。
- 动效不应造成全页噪音。

这些场景可以比普通页面更有“赛事感”，但仍然必须服从全局信息层级。

## 7. 动效策略

本次优化只保留轻量、功能明确的动效：

- 页面区块进入时使用轻微上移或淡入。
- 按钮点击时使用稳定的按压反馈。
- 比赛状态类卡片允许轻度动态强调。
- 弹窗与底部选择器保留顺滑出现效果。

不使用大面积持续动画，不引入与业务无关的炫技效果。

## 8. 实施范围

本次设计覆盖以下前端范围：

- `miniprogram/app.wxss`
- `miniprogram/pages/index/*`
- `miniprogram/pages/activity-list/*`
- `miniprogram/pages/activity-detail/*`
- `miniprogram/pages/create-activity/*`
- `miniprogram/pages/ranking/*`

若在实现中发现已有公共样式可以继续复用，应优先提炼公共规则，而不是继续复制页面私有样式。

## 9. 非目标

本次不包含以下内容：

- 新增业务功能
- 改动云函数接口
- 重写页面业务逻辑
- 引入新的设计库或大型前端框架
- 对全部页面进行彻底重做

若实现过程中发现局部结构必须做少量调整，可在不改变业务语义的前提下适度处理，但不得扩大为无关重构。

## 10. 测试与验收

验收标准如下：

- 首页、列表页、详情页、创建页、排名页风格明显统一。
- 主按钮、次按钮、危险按钮、赛事按钮规则一致。
- 状态标签在不同页面表达一致。
- 卡片样式统一，间距和层级稳定。
- 页面在手机常见尺寸下不出现明显拥挤、错位或视觉失衡。
- 比赛页和排名页保留竞技感，但不破坏全局工具化风格。
- 现有业务流程不受影响，包括登录、创建活动、报名、开始比赛、查看排名等主要路径。

## 11. 实施建议

实现时建议按以下顺序推进：

1. 先统一 `app.wxss` 中的变量、卡片、按钮、标签和基础工具类。
2. 再改首页与活动列表页，先建立全局基准观感。
3. 然后改活动详情页与创建活动页，统一复杂页面的信息组织。
4. 最后调整排名页与比赛卡片，收口竞技氛围场景。

这个顺序可以先稳住全局基调，再处理最具个性的赛事展示页面，减少返工。
