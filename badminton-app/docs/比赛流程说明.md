## 活动状态流转

```
报名中 (registering)
    ↓
小组赛 (playing)
    ↓
挑战赛 (challenge)
    ↓
决赛 (final)
    ↓
结束 (finished)
```

## 各阶段说明

### 1. 报名中 (registering)
- 用户可以报名参加活动
- 组织者可以删除活动
- 报名人数达到要求后，组织者点击"开始分组"

### 2. 小组赛 (playing)
- 系统自动生成小组赛比赛
- 每场比赛需要组织者点击"记分"开始
- 记分流程：待开始 → 比赛中 → 待确认 → 已结束
- **所有小组赛结束后**，组织者点击"开始挑战赛"按钮

### 3. 挑战赛 (challenge)
- 系统自动创建挑战赛（排名第1、2的队伍 vs 淘汰的队伍）
- 组织者点击"开始所有挑战赛"按钮开始所有比赛
- 挑战赛进行中可以进行记分
- **所有挑战赛结束后保持 challenge 状态**，等待创建决赛

### 4. 决赛 (final)
- 挑战赛全部结束后，组织者点击"开始决赛"按钮
- 系统自动创建决赛（第1名 vs 第2名）
- 决赛进行中可以进行记分
- **决赛结束后自动变为结束状态**

### 5. 结束 (finished)
- 所有比赛结束
- 可以查看最终排名

## 组织者操作按钮

| 阶段 | 按钮 | 功能 |
|------|------|------|
| 报名中 | 开始分组 | 生成小组赛 |
| 小组赛 | 开始挑战赛 | 创建挑战赛 |
| 挑战赛 | 开始所有挑战赛 | 开始所有挑战赛 |
| 挑战赛(全部结束) | 开始决赛 | 创建决赛 |
| 决赛 | - | 决赛结束后自动结束 |

## 比赛记分流程

每场比赛的状态流转：

```
pending (待开始)
    ↓ [点击"记分"按钮]
playing (比赛中)
    ↓ [提交比分]
confirming (待确认)
    ↓ [双方确认或组织者确认]
confirmed (已结束)
```

## 积分规则

- 小组赛胜者：+10分
- 挑战赛胜者：+10分
- 决赛胜者：+15分

## 如何恢复头像方案

等头像恢复后（云存储问题解决），按以下步骤操作：

### 步骤1：修改活动详情页 (activity-detail.wxml)

找到文件：`miniprogram/pages/activity-detail/activity-detail.wxml`

找到第 63-68 行（报名列表头像部分），将：
```xml
<!-- 暂时禁用头像，统一显示首字母，等头像恢复后取消注释 -->
<!-- <image wx:if="{{item.avatar}}" class="registration-avatar" src="{{item.avatar}}" mode="aspectFill"></image> -->
<view class="avatar-text">{{item.nickname && item.nickname.length > 0 ? item.nickname[0] : '?'}}</view>
```

恢复为：
```xml
<image wx:if="{{item.avatar}}" class="registration-avatar" src="{{item.avatar}}" mode="aspectFill"></image>
<view wx:else class="avatar-text">{{item.nickname && item.nickname.length > 0 ? item.nickname[0] : '?'}}</view>
```

### 步骤2：修改挑战赛页面 (challenge.wxml)

找到文件：`miniprogram/pages/challenge/challenge.wxml`

找到第 41-46 行，将：
```xml
<!-- 暂时禁用头像，统一显示首字母 -->
<!-- <image wx:if="{{item.avatar}}" class="player-avatar" src="{{item.avatar}}" mode="aspectFill"></image> -->
<view class="avatar-text">{{item.nickname && item.nickname.length > 0 ? item.nickname[0] : '?'}}</view>
```

恢复为：
```xml
<image wx:if="{{item.avatar}}" class="player-avatar" src="{{item.avatar}}" mode="aspectFill"></image>
<view wx:else class="avatar-text">{{item.nickname && item.nickname.length > 0 ? item.nickname[0] : '?'}}</view>
```

### 步骤3：确认其他页面
其他页面如分组页面和排名页面本身就有完整的头像判断逻辑：
```xml
<image wx:if="{{item.avatar}}" class="registration-avatar" src="{{item.avatar}}" mode="aspectFill"></image>
<view wx:else class="avatar-text">{{item.nickname && item.nickname.length > 0 ? item.nickname[0] : '?'}}</view>
```
目前由于头像服务不可用，这些页面都在显示昵称的第一个字符（或'?'如果昵称为空）。
等头像功能恢复后（云存储问题解决），这些页面将自动切换为显示头像（当有头像时）或首字母（当无头像时），无需修改代码。

### 步骤4：重新编译测试

完成以上修改后，重新编译小程序测试头像功能是否正常。
